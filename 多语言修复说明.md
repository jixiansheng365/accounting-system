# 多语言修复说明

## 问题描述
用户反馈：选择中文或繁体中文后，左侧菜单功能区仍然显示英文，没有全部切换为选定的语言。

## 问题原因

### 1. `get_locale()` 函数逻辑问题
原来的 `get_locale()` 函数优先检查 URL 参数，而不是 session 中保存的语言设置。这导致：
- 当用户点击语言切换器（`?lang=zh_TW`）时，语言会正确设置
- 但当用户点击菜单链接时，URL 中没有 `lang` 参数，`get_locale()` 会回退到浏览器语言或默认语言
- 结果就是菜单显示回英文

### 2. 修复方案
修改 `get_locale()` 函数的优先级：
1. **优先检查 session**（最高优先级）
2. URL 参数
3. 浏览器 Accept-Language 头
4. 默认语言（简体中文）

## 修复内容

### 修改文件：`app/__init__.py`

#### 修改前：
```python
def get_locale():
    # 支持的语言列表
    supported_langs = ['zh_CN', 'zh_TW', 'ja', 'en', 'ko']
    
    # 检查 URL 参数
    lang = request.args.get('lang')
    if lang in supported_langs:
        session['lang'] = lang
        return lang
    
    # 检查 session
    if 'lang' in session:
        return session['lang']
    
    # 检查浏览器语言
    browser_lang = request.accept_languages.best_match(...)
    # ...
```

#### 修改后：
```python
def get_locale():
    # 支持的语言列表
    supported_langs = ['zh_CN', 'zh_TW', 'ja', 'en', 'ko']
    
    # 1. 优先检查 session 中的语言设置
    if 'lang' in session and session['lang'] in supported_langs:
        return session['lang']
    
    # 2. 检查 URL 参数
    lang = request.args.get('lang')
    if lang in supported_langs:
        session['lang'] = lang
        return lang
    
    # 3. 检查浏览器语言
    browser_lang = request.accept_languages.best_match(...)
    if browser_lang in supported_langs:
        session['lang'] = browser_lang
        return browser_lang
    
    # 4. 默认语言为简体中文
    return 'zh_CN'
```

## 修复效果

### 修复前：
1. 用户点击"繁" → 页面显示繁体中文 ✅
2. 用户点击菜单"Dashboard" → 菜单变回英文 ❌

### 修复后：
1. 用户点击"繁" → 页面显示繁体中文 ✅
2. 用户点击菜单"Dashboard" → 菜单保持繁体中文 ✅
3. 所有后续页面都保持用户选择的语言 ✅

## 注意事项

1. **Session 持久性**: 语言设置保存在 session 中，关闭浏览器后会失效
2. **URL 参数**: 所有内部链接都应该携带 `lang` 参数以保持一致性
3. **翻译完整性**: 确保所有新添加的文本都有对应的翻译
4. **编译翻译**: 修改 `.po` 文件后需要重新编译为 `.mo` 文件
