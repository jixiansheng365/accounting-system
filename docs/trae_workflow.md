## 各智能体职责

### 1. 架构师智能体（总控 + 终审）

**职责**：
- 制定并下发《统一开发规范档案库》
- 审批所有技术方案
- 终审代码，通过才算完成
- 把控技术方向，防止跑偏

**输出文档**：
- `docs/architecture.md` - 架构设计
- `docs/tech_stack.md` - 技术栈规范
- `docs/ui_standards.md` - UI 统一规范
- `docs/api_spec.md` - API 规范

**审批节点**：
- [ ] 技术方案审批
- [ ] UI 设计审批
- [ ] 代码终审

---

### 2. UI 工程师智能体

**职责**：
- 严格按照《UI 统一规范》设计
- 使用标准色值：#0d6efd / #6c757d / #28a745
- 使用标准布局：modern-box + modern-title
- **禁止自定义风格**

**设计检查清单**：
- [ ] 是否使用 `modern-box` 卡片？
- [ ] 是否使用 `modern-title` 标题？
- [ ] 颜色是否符合标准？
- [ ] 间距是否统一（24px/12px/16px）？
- [ ] 圆角是否统一（12px）？
- [ ] 是否有 Font Awesome 图标？

**禁止事项**：
- ❌ 自定义颜色
- ❌ 自定义间距
- ❌ 自定义字体
- ❌ 自定义动画

---

### 3. 开发智能体

**职责**：
- 按照规范写代码
- 所有页面统一布局
- 遵循 MVC 架构
- 使用标准组件
- **【强制】每次修改必须检查路由链接**

**开发检查清单**：
- [ ] 是否使用 `container-fluid px-3 px-md-4`？
- [ ] 是否使用 Bootstrap5 类？
- [ ] 是否使用 DataTables 表格？
- [ ] 是否有返回按钮？
- [ ] 底部是否有默认头像？
- [ ] 业务逻辑是否完好？
- [ ] **【强制】所有路由链接是否正确？**
- [ ] **【强制】url_for 引用的路由名是否存在？**
- [ ] **【强制】新添加的路由是否同时支持必要的 HTTP 方法？**

**路由检查规则（强制）**：
```
1. 添加新页面路由时，必须同时检查：
   - 路由路径是否正确
   - 是否支持 GET 方法（页面访问）
   - 是否支持 POST 方法（表单提交）
   
2. 修改现有路由时，必须检查：
   - 所有引用该路由的模板是否需要更新
   - url_for 中的路由名是否正确
   
3. 常见错误对照表：
   - ❌ url_for('admin.logout') → ✅ url_for('admin.admin_logout')
   - ❌ url_for('admin.reports') → ✅ url_for('admin.reports_page')
   - ❌ url_for('admin.customers') → ✅ url_for('admin.customers_page')
   - ❌ url_for('admin.upload') → ✅ url_for('admin.upload_page')
   - ❌ url_for('auth.login') → ✅ url_for('auth.login_page')
   
4. 路由修改后必须：
   - 全局搜索所有引用该路由的文件
   - 更新所有相关模板中的链接
   - 测试访问是否正常
```

**代码规范**：
- 前端：Bootstrap5 + jQuery + DataTables
- 后端：Flask + SQLAlchemy
- 模板：Jinja2
- 样式：基于 dashboard.css 扩展

**禁止事项**：
- ❌ 不检查路由链接就提交代码
- ❌ 修改路由后不更新模板引用
- ❌ 只注册 POST 路由却期望 GET 访问

---

### 4. 测试智能体

**职责**：
- 按照规范验收
- 发现风格不统一直接反馈
- 验证功能完整性
- 测试响应式布局

**验收检查清单**：
- [ ] 颜色是否统一？
- [ ] 布局是否统一？
- [ ] 间距是否统一？
- [ ] 字体是否统一？
- [ ] 图标是否统一？
- [ ] 交互是否统一？
- [ ] 响应式是否正常？

**反馈标准**：
- 发现风格不统一 → 立即标记
- 发现布局混乱 → 立即退回
- 发现技术栈违规 → 立即上报

---

### 5. 项目经理智能体

**职责**：
- 协调各智能体工作
- 推动整改，确保全系统统一
- 跟踪任务进度
- 处理阻塞问题

**工作流程**：
1. 接收架构师下发的规范
2. 分配给 UI 工程师设计
3. 分配给开发智能体实现
4. 提交测试智能体验收
5. 如不达标，推动整改
6. 提交架构师终审
7. 通过后合并发布

**整改推动**：
- 收到测试反馈 → 立即分配给相关智能体
- 跟踪整改进度 → 确保按时完成
- 整改后重新测试 → 直至通过
